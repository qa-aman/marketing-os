# Documents needed to run the workshop

Everything we need to build, what each is for, who uses it, and its status. Build order follows the phases in the README.

## The core set

| # | Document | What it is / why | Who uses it | Status |
|---|---|---|---|---|
| 1 | Facilitator guide (run of show) | The master script: every part, timing, talking points, what to say, when to demo, when to switch to a lab. The one doc that runs the day. | You (instructor) | not started |
| 2 | Slide deck | The visual backbone the room follows. One short section per capability, screenshots, the Verdal examples. | Room | not started |
| 3 | Demo scripts | For each capability, the exact clicks and the exact copy-paste prompts on Verdal, so live demos never fail. One script per Part. | You (instructor) | not started |
| 4 | Hands-on lab worksheets | What attendees do on their own laptops each Part, step by step, on Verdal. This is where learning sticks. | Attendees | not started |
| 5 | Pre-workshop setup sheet | Sent 2-3 days before. Prerequisites: a paid Claude subscription with Cowork, install steps, download the Verdal folder, optional connector accounts. Stops day-one setup chaos. | Attendees | not started |
| 6 | Capability cheat sheet | A one-pager the team keeps at their desk: the 17 capabilities, the trigger, one example each. | Attendees (after) | not started |
| 7 | Knowledge-check quiz | Mirrors Anthropic Module 5. Confirms the team absorbed the core ideas. | Attendees | not started |
| 8 | Feedback form | Short post-workshop survey: what landed, what to fix, what they will use first. | Attendees | not started |
| 9 | Facilitator FAQ / troubleshooting | Common failure points (no folder access, connector not authorized, wrong permission mode) and the fix, so nobody gets stuck. | You (instructor) | not started |

## Optional extras (nice to have)

| Document | Why |
|---|---|
| Recorded demo clips | 2-3 minute screen recordings of the hardest demos, as a backup if a live demo fails or for people who miss the session. |
| Self-paced version | Convert the workshop into tutor-style commands (like the marketing workshop) so new joiners can take it later without a live session. |
| Team skill starter pack | A small set of ready-made Verdal skills attendees can install on day one, so they leave with working tools, not just knowledge. |

## Build sequence (which document, which phase)

- Phase 2: documents 1 and 2 (facilitator guide, slide outline).
- Phase 3: documents 3 and 4 (demo scripts, lab worksheets).
- Phase 4: documents 5, 6, 7, 8, 9 (setup sheet, cheat sheet, quiz, feedback, FAQ).
- Phase 5: dry run, then the optional recorded clips if wanted.
- Phase 6: after delivery, the optional self-paced version.

## Definition of done

A colleague who has never run this could pick up the facilitator guide, the slides, the demo scripts, and the setup sheet, and deliver the full workshop without asking us a single question.
