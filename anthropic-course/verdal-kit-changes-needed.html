Verdal kit - readiness audit

What to add to the verdal-marketing folder, and why

An end to end check: every "Paste into Cowork" prompt in the six guides, run against the actual files in the kit. Where a marketer would paste a prompt and Cowork could not respond properly, this report says exactly what to add, why it is needed, and what value it brings.

Scope: anthropic-course/verdal-marketing/ and its six guides/. Six changes below, ordered by priority.

Resolved. All six changes below are now applied to the kit and re-audited against the real files. Every prompt passes. This report is kept as the record of what was changed and why. See the After-fix column in the scorecard.
The short version. Of 10 concrete prompts across the guides, 6 worked as they stood. 4 needed a fix: three broke on missing data (Sessions 2, 3, 6), and one needed a connector with no offline fallback (Session 4). None required rewriting the guides. Each was a small file or folder added to the kit, now done.

The verdict

A non-technical marketer loads the kit, opens a guide, and pastes prompts. Here is what happens today:

  • Works (6): Session 1 captions, Session 1 strategy deck, Session 1 homework email, Session 5 live-artifact dashboard, Session 5 parallel research, Session 5 scheduled digest (from local fallback).
  • Breaks on missing data (3): Session 2 carousel project, Session 3 package-the-workflow, Session 6 validate-the-skill.
  • Needs a connector, no fallback (1): Session 4 Gmail task. Plus one inherently-live Chrome task that would benefit from a worked example.

The three "breaks" are the important ones: they are the flagship hands-on moments of their sessions, and they fail on the first step.

CHANGE 01 High

Add a real carousel-design-system project folder

What to add
verdal-marketing/carousel-design-system/ 
carousel-design-system/
  context/                  copy of the three brand context files
  reference/
    carousel-examples.md    copy of reference/carousel-examples.md
  output/                   empty, the project saves carousels here
  README.md                 what this project is for
Unblocks

Session 2, the carousel project build. The guide tells the marketer to "create a project from your carousel-design-system folder."

Why it is needed

Today the kit has only a file, templates/carousel-design-system.md, not a folder. When the marketer goes to pick the folder, there is nothing to select. Session 2's flagship "big win" exercise fails on the first click.

Value

Session 2 becomes runnable exactly as written, with no edit to the guide. The marketer sees a real project with its own context and reference, which is the whole point of the projects lesson.

CHANGE 02 High

Ship the weekly-content-brief skill file

What to add
verdal-marketing/templates/weekly-content-brief/SKILL.md 
---
name: weekly-content-brief
description: Use when asked for the weekly Verdal content brief.
---
Read context/. Pick three topics for Plant-proud Priya this
week. Write a short brief for each (angle, hook, format).
Save to projects/weekly-content-brief.md. Follow brand-voice.md.
Unblocks

Session 6, validate and share the skill. Its own prerequisite callout says this file should already be in the kit.

Why it is needed

Session 6 opens by running weekly-content-brief, but no skill file exists anywhere in the kit. If a marketer did not complete Session 3, or saved the skill to Cowork's own panel instead of this folder, Session 6 has nothing to run.

Value

Session 6 works standalone, independent of whether Session 3 succeeded. It also gives Session 3 a correct reference to check their own version against.

CHANGE 03 High

Seed a sample "plant-care trend" email

What to add
verdal-marketing/reference/plant-care-trend-email.md 
A realistic email a teammate might send, for example:
"Adobe's plant-care category report - winter watering
searches up 40%, 'why are my leaves yellow' is the top
query, people want less-is-more advice." Two or three
paragraphs, so Cowork has something real to pull from.
Unblocks

Session 4, the Gmail live-data task ("find the latest plant-care trends email and add a this-month section to the newsletter").

Why it is needed

Verdal is fictional, so no attendee's real inbox has this email. With no seeded source, the exercise either stalls or invents a "this month" section from nothing, which defeats the "read a live source" lesson.

Value

Every attendee can complete the exercise offline by pointing Cowork at the file. Facilitators who want the full live version get a ready script to plant in a demo inbox. The guide gets a one-line note: "no Gmail? point Cowork at reference/plant-care-trend-email.md instead."

CHANGE 04 Medium

Add a prior weekly-brief example

What to add
verdal-marketing/projects/weekly-content-brief-example.md 
A finished example of the weekly content brief: three
topics for Priya with angle, hook, and format each,
in Verdal's voice. This is the "workflow we just ran"
that Session 3 asks the marketer to package.
Unblocks

Session 3, package-the-workflow. The prompt says "the weekly content brief workflow we just ran."

Why it is needed

Nothing earlier in the kit shows that workflow, so Cowork has to improvise what it is packaging. Results vary run to run and the lesson lands weakly.

Value

The skill-creation demo becomes consistent and repeatable. The skill-creator has a concrete example to generalize from, instead of inventing one on the spot.

CHANGE 05 Low

Add a worked search-audit example

What to add
verdal-marketing/reference/search-audit-example.md 
A mock of what a finished AI-search audit looks like:
the three prompts checked, whether Verdal appeared, the
top brands cited, and one gap. So people without Claude
in Chrome can still see the expected shape.
Unblocks

Session 4, the Claude in Chrome task (a live-web task with no full offline substitute).

Why it is needed

Participants without Chrome enabled have nothing to compare their attempt against, and the facilitator cannot demo the output shape if the connector is not working in the room.

Value

The session still teaches the idea and the output format even when the live connector is unavailable.

CHANGE 06 Low

Put the spare data files to work (or label them)

What to change

Two of the four data files, data/last-month-metrics.csv and data/content-planner.csv, are never named by any guide. Either add a prompt that uses them, or note in the kit README that they are bonus material for self-directed practice.

Why it is needed

Right now half the data/ folder is decorative. A facilitator opening the folder wonders what they are for.

Value

Nothing is wasted. Either the data earns its place in a lab (a quick KPI review or a content-calendar task), or people know it is optional practice fuel.

Suggested use: a Session 5 or bonus prompt like "Read data/last-month-metrics.csv and tell me which channel to double down on next month, with one reason."

How to apply

All six changes are additive. Nothing in the kit needs to be removed, and no guide needs rewriting. Priority order:

  1. High: the carousel project folder, the weekly-brief skill file, the sample trends email. These unblock three broken labs.
  2. Medium: the weekly-brief example output.
  3. Low: the search-audit example, and putting the two spare CSVs to use.
After adding these, the kit is fully self-contained. Every prompt in every guide will produce a real result from local data, with connectors becoming an optional upgrade rather than a requirement. That matters for a room where not everyone has logins and tools wired up on day one.
