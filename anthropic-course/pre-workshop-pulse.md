# Pre-workshop pulse - WhatsApp polls

A 60-second read for the team before the Claude Cowork workshop. The point is to learn where they are starting from, so we can pace the sessions and sort logins. It is not a test. Frame it that way, or people will not answer.

> Prefer a Google Form that saves every answer to a sheet? Run `create-pulse-form.gs` (instructions inside). It builds two forms: `createPreForm()` for before the workshop (readiness plus the capability self-map) and `createPostForm()` for after (the same capability self-map plus feedback). The gate and capability questions are identical in both, so you compare those two sheets to measure what the team learned.

Aditya posts these in the company group. Use WhatsApp's native Poll feature (tap the plus, then Poll) for each one, so people answer with a single tap and we get clean counts. Five short polls, about 50 seconds total.

## The two polls that answer "do they already know this?"

These two map straight to the course, so you learn whether the team already knows the steps we are about to teach. Post these along with the five below.

### Poll A - The gate: have they used Cowork at all?

**Question:** Have you used Claude Cowork, the desktop agent that works with your files (not the Claude chat)?
- Yes, I use it
- Tried it once or twice
- No, but I know what it is
- No, it is new to me

*Read it like this: anyone who has not used Cowork does not yet know the steps, by definition. If most say "new to me," the workshop is pitched right. If a chunk say "I use it," we may add a faster or deeper track for them.*

### Poll B - The capability self-map (allow multiple answers)

**Question:** Which of these can you already do in Claude Cowork today? Tap all that apply.
- Give it a task on a folder and get a finished file back (Session 1)
- Set brand rules or a project it remembers (Session 2)
- Build a reusable skill (Session 3)
- Connect a tool like Gmail or Drive (Session 4)
- Build a live dashboard or schedule a task (Session 5)
- None of these yet

*Turn on "allow multiple answers" for this poll. The count on each option tells you, session by session, how many already know that content. Heavy "none of these yet" confirms the level is right. If one option scores high, we compress or level up that session.*

## What each answer buys us

| Poll | Tells us | How it changes the workshop |
|---|---|---|
| A | Whether they have used Cowork at all | The clearest read on "do they already know this". Mostly "new to me" means the level is right. Some "I use it" means add a faster track for them. |
| B | Per session, how many already know that content | Session by session: heavy "none yet" confirms the level; a high-scoring option means compress or level up that session. |
| 1 and 2 | Overall AI and Claude maturity | Sets pace and depth. Lots of "never" means slow down and over-explain; lots of "daily" means we can move faster and go deeper. |
| 3 | Learning style | How much hand-holding to build in, whether to add more "watch me first" before labs. |
| 4 | Setup readiness | The big one. If many are on locked machines or need IT approval, we sort that before Session 1 instead of losing the first session to install problems, and we decide individual vs shared logins. |
| 5 | What they care about | Which demos to lead with, so the sessions feel relevant to their real work. |

## A few tips

- Keep it to these five. More polls, fewer answers.
- Post all five close together so it reads as one quick task.
- WhatsApp polls show who voted, so keep the questions low-stakes (these are). If you want fully anonymous, a Google Form works but expect a lower response rate.
- Aim to send it 3 to 5 days before Session 1, so there is time to fix any setup or IT issues Poll 4 surfaces.
- Even 20 of 30 responses gives a clear enough read to calibrate.
