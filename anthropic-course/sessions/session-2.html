Global instructions vs projects

Same idea, different scope.

Global rules apply to every task. A project is a dedicated workspace for one workflow you run again and again.
Global instructions

Rules for every task, every session, every folder.

voice, folder conventions, output format
Projects

One repeatable workflow, with its own memory and context.

monthly carousel, weekly email, quarterly report
