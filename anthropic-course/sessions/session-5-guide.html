Bigger outputs and automation

So far you have made one thing at a time: a brief, a draft, a report. Each one was useful, then done the moment it finished. Today is different. You build outputs that stay alive: a dashboard that refreshes itself, and a job that runs every week without anyone pressing a button.

Worked brand: Verdal, a plant care brand. Dataset: verdal-marketing/. Official course: Introduction to Claude Cowork.

What you need before you start. Claude Desktop installed and signed in, with Cowork in the sidebar. The verdal-marketing folder loaded. Artifacts and Code Execution turned on in Cowork settings (Preferences - Features). If you skip that setting, none of today's builds will work.

Build things that keep working

Sessions 1 through 4 covered single tasks: one brief, one draft, one report. You wrote a prompt, Cowork produced a file, and that was the end of it. Today the output does not end when the file is saved.

Two ideas that change how you use Cowork

A static fileA live artifact
What it isA finished documentA small interactive dashboard
How it updatesYou re-run the promptYou click refresh
Where it livesprojects/Pinned in your sidebar

Add a scheduled task on top: a job you set up once, that Cowork runs on its own every week and saves into your folder. Put the two together and you get a control center: the scheduled task fills the data, the live artifact shows it, and you just open it on Monday morning.

STEP 01

Live artifacts: a dashboard that refreshes

Build a Verdal dashboard that pulls fresh keyword and competitor data every time you click refresh, not just once.

Do this

  1. Open Cowork settings (Preferences - Features) and turn on Artifacts and Code Execution. Both must be on.
  2. Use the exact words "live artifact" somewhere in your prompt. This is the phrase that tells Cowork to build something interactive instead of a plain file.
  3. Paste the prompt below into Cowork.
  4. When it appears, pin it to your sidebar.
  5. Click refresh and watch the "last refreshed" timestamp update.
Paste into Cowork
the Verdal dashboard 
Load verdal-marketing/. Build a LIVE ARTIFACT showing this
week's keyword opportunities and competitor activity for
Verdal, using data/keyword-opportunities.csv and
data/competitor-posts.csv. Pin it and show a "last refreshed"
timestamp.
Say the words, or you get a static file. If your prompt says "dashboard" without the phrase "live artifact," Cowork will write a plain markdown file that never updates. The words "live artifact" are what turn it into something you can refresh.

What good looks like

  • A dashboard appears with a chart of keyword opportunities and a competitor activity summary, pulled from the two CSV files.
  • A pin option is visible, usually in the top-right corner of the artifact panel.
  • A "last refreshed" timestamp is shown somewhere on the dashboard.

Test it

Pin the artifact, then click refresh. If the timestamp changes but nothing else happens, that is fine, it means it is reading the same data again. If refresh does nothing at all, check that Artifacts and Code Execution are both switched on in settings, then try again.

STEP 02

Scheduled tasks: set a recurring job

Some work should just happen. Set it up once, and Cowork runs it on the schedule you pick, saving the result into your folder every time.

Do this

  1. Open the scheduled tasks panel in Cowork.
  2. Give it a name: Verdal weekly competitor digest.
  3. Paste the prompt below as the task.
  4. Set the folder to verdal-marketing/projects/.
  5. Set the frequency to every Sunday at 9am.
  6. Confirm the status shows Active.
Paste into Cowork
the scheduled digest 
Every Sunday at 9am, research Verdal's competitors and
write a short weekly competitor digest. Save it to
projects/weekly-digest.md.

What good looks like

  • The task appears in the scheduled tasks panel with the right name, folder, frequency, and status Active.
  • Recurring, predictable, data-driven work is what belongs here: a weekly content brief, a weekly competitor digest, a monthly performance report. Save creative strategy and one-off work for prompted sessions.

Test it

If the task does not show up in the panel after saving, close and reopen the project in Cowork. That refresh usually surfaces it.

STEP 03

The control center

A live artifact on its own is useful. A scheduled task on its own is useful. Together, they run themselves.

Here is how the two connect:

how it connects 
Sunday 9am: scheduled task runs
  -> writes fresh data to projects/weekly-digest.md
Monday morning: you open the pinned dashboard
  -> live artifact reads the new file
  -> you click refresh and see this week's numbers

You set this up once. From then on, you just open it. That is the marketing control center: the scheduled task fills the data, the live artifact surfaces it, and nobody has to re-prompt anything.

Try it end to end. Point your live artifact from Step 01 at projects/weekly-digest.md, the file your scheduled task from Step 02 writes to. Next Sunday, the file updates on its own. Next time you open the dashboard, refresh pulls it in.
STEP 04

Parallel and research workflows

For big research jobs, Cowork can work on several things at once instead of one at a time.

Use the phrase "research [targets] in parallel" and Cowork spins up a helper for each target, then merges the findings into one report. For a competitor scan across three brands, that is three research jobs running at the same time instead of three separate prompts run back to back.

Paste into Cowork
parallel competitor research 
Load reference/competitors.md. Research The Sill,
Bloomscape, and Leon and George in parallel. Merge the
findings into a single competitor summary and save it
to projects/competitor-summary.md.

What good looks like

  • The to-do list shows separate helper tasks running for each competitor, marked done as they finish.
  • A final "merge findings" step combines them into one report.
  • You get three competitors' worth of research in roughly the time of one.

Test it

If Cowork researches the three targets one after another instead of at the same time, add "in parallel" more explicitly, for example: "research these three in parallel, not one at a time."
