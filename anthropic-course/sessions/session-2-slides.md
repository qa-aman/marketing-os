## Slide 5 - Global instructions: what they are
**On screen:**
- Rules you set once, applied to every session.
- Write them in Settings. Cowork reads them before every task - no matter the folder, no matter the project.
- Right panel - code block showing example global instructions: read context/, save to projects/, match brand-voice.md, no em dashes, warm expert plain tone.

**Notes:** Global instructions are a short block of text you write once in Settings. Cowork reads them before every task, in every session, for every folder. Think of it as a standing briefing. Keep it under 100 words: just the rules that should always apply.

## Slide 6 - Where to set global instructions
**On screen:**
- Settings, then one text field.
- Settings > Cowork > Global instructions. Type it once. It runs before every task from here on.
- One or two lines is enough. Save the detail for project instructions.
- Right panel - settings navigation tree with a text area showing example instructions.

**Notes:** Show this in the live app. Settings is the gear icon, bottom left. Global instructions is one text field. Type it once, save. Done. Remind them: keep it short. One or two lines is enough if that is all that needs to apply globally. Save the detail for project instructions.

## Slide 7 - Projects
**On screen:**
- A project is a reusable workspace.
- Its own instructions, its own context, its own memory. Set it up once, and every run builds on the last.
- Right panel - a window showing a project folder tree: carousel-design-system/ with project-instructions.md, context/, reference/, output/.

**Notes:** A project is a named workspace inside Cowork. It has its own folder, its own instructions, and its own memory of past work. The key difference from global instructions: a project is for one repeatable workflow, not all tasks.

## Slide 8 - What a project remembers
**On screen:**
- The folder, the rules, and every piece of work it has done.
- Open the project next week. It still knows the brand, the design rules, and what it made last time.
- Right panel - checklist: brand context, project instructions, reference examples, every carousel it has made, feedback from past runs.

**Notes:** This is the part that usually surprises people. A project does not just hold files. It remembers the instructions you gave it, the context you loaded, and the work it has already done. Next time you open it, all of that is still there.

## Slide 9 - Global instructions vs projects
**On screen:**
- Same idea, different scope.
- Global rules apply to every task. A project is a dedicated workspace for one workflow you run again and again.
- Right panel - two green cards: "Global instructions: rules for every task, every session, every folder" and "Projects: one repeatable workflow, with its own memory and context."

**Notes:** This is the clarifying slide. Both are standing context. The difference: global instructions apply to everything, always. A project is a focused workspace for one repeatable workflow. Use global for universal rules, projects for specific recurring work.
