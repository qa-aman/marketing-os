# Session 2 facilitator guide - Make Cowork yours

**Length:** 2 hours. **Audience:** non-technical marketers. **Example:** Verdal. **You need:** the Verdal folder on the demo machine, Cowork installed and signed in, global instructions already typed in Settings, the carousel-design-system project folder ready, this guide, and the slides.

Goal for the room: by the end, every attendee has set at least one global instruction and created one Cowork project with its own instructions. They leave understanding the difference between a rule that applies to everything and a workspace for one workflow.

Capabilities covered: global instructions, projects, project instructions, standing context.

## 0:00 - 0:05  Welcome and recap

Say: "Last session you ran a first task in Cowork and watched it save a finished file. Today is about removing the setup work: you should never have to explain your brand twice. By the end of this session, Cowork will already know your rules before you type a word."

Set the frame: two tools today, global instructions and projects. They sound similar but serve different purposes. You will know which to use for what.

## 0:05 - 0:40  Teach - standing context

Keep this section tight. Use the agenda slide to signal how far you are. Cover, in order:

1. **The problem.** Ask who has copy-pasted brand rules into Cowork at the start of a session. That is the behaviour we are eliminating today. Without standing context, every session starts blank and you re-teach Cowork everything it already knew yesterday.

2. **Global instructions.** A short block of text you write once in Settings. Cowork reads it before every task, in every session, in every folder. It is a standing briefing, not a per-task paste. Keep it short: voice, folder conventions, output format. Under 100 words is plenty.

3. **Where to set them.** Settings is the gear icon, bottom left. Settings > Cowork > Global instructions. One text field. Type it, save it, done. Show this live in the app for 30 seconds. Do not linger.

4. **Projects.** A project is a named workspace inside Cowork. It has its own folder, its own instructions, and its own memory of past work. Unlike global instructions, a project is for one repeatable workflow. The carousel project you will demo has its own brand context, design rules, and a record of every carousel it has made.

5. **What a project remembers.** Brand context, project instructions, reference files, and every piece of output it has produced. Open a project next week and it is still where you left it.

6. **Global vs projects.** The clarifying frame: use global instructions for rules that should apply everywhere - voice, folder conventions, what to avoid. Use a project for a workflow you run again and again, like a monthly carousel or a weekly email. The two work together: global rules apply inside projects too.

Analogy that helps: "Global instructions are like the team handbook every new colleague reads on day one. A project is the dedicated Slack channel for one campaign, with its own pinned context and shared history."

## 0:40 - 1:00  Demo - carousel project from Verdal

This is the show. Follow the demo walkthrough below exactly.

**Before you start:** confirm your Settings > Global instructions are visible and have something in them. The room should see that the setup is real.

**Demo walkthrough:**

1. Open Settings > Cowork > Global instructions. Read the text out loud: "Always read context/ before any task. Match brand-voice.md. No em dashes, no emojis." Say: "I typed this once. Every task I run today - and every task I run next week - starts with this."

2. Close Settings. Open the carousel-design-system project. Show the folder tree: context/ with the brand files, reference/ with example carousel PDFs, and an empty output/ folder.

3. Show the project instructions file (project-instructions.md). Read the key line: "Use warm expert tone. Follow the Verdal carousel design system."

4. Paste the demo prompt: "Create a project from my carousel-design-system folder. Read the reference examples and brand context, then write a brand style guide and a carousel design system file, and save them in the project."

5. Narrate the task loop as it runs: "See it reading brand-context.md first - that is the global instruction working. Now it is reading the reference examples. Now it is writing the style guide."

6. Open the two finished files in output/. Point out the Verdal voice - warm, expert, plain. Point out the green and terracotta from the brand. Say: "I did not paste any of that. It came from the project's context and my global instructions."

If the demo is slow, keep narrating: describe what the task loop is doing. Do not sit in silence. If it fails, switch to the recorded backup clip (build one before the session, same prompt, same folder).

## 1:00 - 1:05  Break

## 1:05 - 1:50  Lab - set your rules, build a project

Attendees work on their own laptops with their own Verdal copy. Walk the room.

High level - three steps:

1. Settings > Cowork > Global instructions. Paste your rules. Keep it short: voice, folder, format. Two or three lines maximum.

2. In Cowork, create a new project from verdal-marketing/context/ or any subfolder they have. Name it clearly - "Verdal email" or "Verdal carousel."

3. Add project instructions inside the project. Then run the starter prompt from the lab slide. Watch the output land in the project's output folder.

Starter prompt (also on the lab slide):
"Create a project from verdal-marketing/context/. Name it 'Verdal email'. Add this project instruction: 'Always read context/ and match brand-voice.md before writing.' Then write a short welcome email for Nate, a new plant owner, and save it to projects/welcome-email.md"

Common blockers and fixes:

- **Project instructions not added** - they skipped step 3. Redirect them: click the project settings or instructions file and type two lines before running.
- **Output in the wrong place** - they are looking in the global chat output instead of the project's output/ folder. Show them where the project folder lives on disk.
- **Global instructions empty** - they went straight to the project. Ask them to go back to Settings first. Two minutes, not a big deal.

Login note: if some attendees share a common login, they follow along while the volunteer drives and do the real task as homework. Confirm this before the session.

## 1:50 - 1:55  Debrief and Q&A

Ask three questions. Let a few people answer each.

- What will you set as a global rule? (This surfaces what matters most to them.)
- What did the project remember between runs? (This confirms they saw it work.)
- Where do projects save their output? (This catches the most common confusion.)

The goal: they can articulate in their own words why global instructions and projects are different tools.

## 1:55 - 2:00  Preview and homework

Preview Session 3: "Next time we look at skills and plugins. You will package a workflow so Cowork repeats it the same way every time, on any brand, without you re-explaining the steps."

Homework (can be done in five minutes): set one global instruction in Settings, then create one project on your own brand's folder, add a project instruction, run one task, and note what it saves and where. Bring the output to Session 3.

## Knowledge check (send 1-2 days after the session)

- Q: What is the difference between global instructions and a project? A: Global instructions apply to every session and every folder; a project is one reusable workspace with its own instructions and memory.
- Q: Where do you set global instructions? A: Settings > Cowork > Global instructions.
- Q: What does a project remember between sessions? A: Its brand context, project instructions, reference files, and every piece of output it has produced.

Send these by message or a quick form 1-2 days after the session. Retrieval a day or two later is what makes it stick.
