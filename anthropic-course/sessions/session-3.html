What a skill is

A saved workflow with a name and a description.

The description is the trigger. Cowork reads it to know when to use the skill.
weekly-content-brief / SKILL.md
name: weekly-content-brief description: use when asked for the weekly Verdal content brief --- Read context/, pick 3 topics that fit Plant-proud Priya this week, write the brief, save to projects/.

Before you build a skill

Should this even be a skill?

Build one when the answer is yes to two of these. Otherwise just prompt it.
  • Have you done this 3+ times?
  • Would you train a new hire on it?
  • Must the quality be the same every time?
Turn it on first

Two switches before skills work.

In Settings, Capabilities: turn on code execution and file creation. Then enable skills one at a time, not all at once.
Settings → Capabilities
Code execution   on
File creation   on
then switch on the one skill you need
