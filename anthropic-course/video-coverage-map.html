What to add, and where (Cowork, in scope)

Everything below runs in the desktop app with no code. Each item names the session file it belongs in.

Session 1 - Meet Cowork (sessions/session-1.html)

  • The three modes in one slide - Chat (cloud, hands you text), Cowork (local, does the work), and where Claude Code sits (power users, out of scope here). From videos 6, 10.
  • Point at a folder, do not paste into chat - the habit that makes Cowork better than Chat. From video 6.

Session 2 - Standing context & projects (sessions/session-2.html)

  • Let Claude interview you to build your brand files - point Cowork at an empty folder and ask it to create brand-context.md, brand-voice.md, and ideal-customer-profile.md by asking you questions, then save them to context/. The fastest way for the team to onboard their real brand after the workshop. From videos 1, 12.
  • Set the project instructions field for every project (the per-project system prompt), separate from the context files. From videos 1, 12.

Session 3 - Skills & plugins (sessions/session-3.html)

  • The "should this be a skill?" rule - build one when a task repeats 3+ times, you would train a human on it, and quality has to be consistent. From video 8.
  • Enable code execution + file creation first, then turn on skills one at a time (not all at once). From video 8.
  • Two ways to make a skill - use skill-creator, or extend an official skill so you do not start from a blank page. Tell Claude to make it brand-agnostic. From videos 5, 6, 8.
  • Install the official Anthropic marketing plugin for working skills on day one (brand review, campaign plan, competitive brief). From video 1, 6. (official plugin, install docs)

Session 4 - Connect your tools (sessions/session-4.html)

  • A connector in one breath + the safety order: native > official > community > custom, and grant the minimum access. From video 10.
  • Notion as the worked example - connect it, push a content calendar, build a dashboard from a tracker. Pairs with the distribution recipe. From video 9.
  • Apify - full segment: what web scraping is, install it as a connector, scrape competitor social posts into a spreadsheet + dashboard for Verdal. From video 1. (apify.com, docs)

Session 5 - Bigger outputs & automation (sessions/session-5.html)

  • Gamma - full segment: connect Gamma inside Claude to turn Cowork's work into a polished, editable deck; a Verdal example. From video 1. (help.gamma.app)
  • Live-artifact gotchas: turn on Artifacts + AI-powered artifacts + Code Execution first; use the literal words "live artifact" in the prompt; pin it and hit refresh to update without rerunning. From video 1.
  • Parallel agents + self-tracking for big jobs - tell Cowork to run parts in parallel and keep its own checklist so nothing is dropped, then spot-check. From video 6.
  • Research with Perplexity, build with Claude - the research-to-deliverable flow: Perplexity gathers live sources, Claude turns them into a dashboard, competitor report, or persona guide. Two web tools, no code. From video 13.
  • Optional native-connector starter pack - a research/Drive agent and a Gmail+Calendar+Drive ops agent, as Projects. From video 12.

Session 6 - Safety, sharing, next steps (sessions/session-6.html)

  • Human-in-the-loop for high-stakes actions - any agent that sends or publishes must confirm first; never give send + delete together. From videos 7, 12.
  • Use two models together - Gemini for scale and long documents, Claude for strategy and story, and the critique loop where one drafts and the other sharpens it. Non-technical, just two web tools. From video 11.
  • The "beyond Cowork" pointer - one honest slide: this is where Claude Code and agent teams begin, for anyone who wants to go further.

Excluded - Claude Code only (too technical)

Only one tool is left out: Claude Code. Videos 2, 3, 5 and 7 (and the technical setup parts of 10) need a terminal, JSON config, or command-line agent tools, which breaks the promise that this workshop needs no code. They are named here, in one place, so nobody assumes they were missed. A one-slide "beyond Cowork" pointer in Session 6 tells the team this ceiling exists, without teaching it.

  • Building a multi-agent marketing team in Claude Code (subagents, /agents, routing via CLAUDE.md). Videos 3, 7.
  • Skills at scale in Claude Code: brand/function/specialty taxonomy, Agent Teams, a Notion skill library. Videos 2, 5.
  • MCP the technical way: local JSON config, community servers, no-code n8n custom servers. Video 10 (the native-connector half is taught in S4).

Note: Gemini and Perplexity are not excluded. They run in the browser with no code, so their marketing workflows (videos 11 and 13) are taught in the Advanced phase.

Status is judged against the non-technical, Cowork-only scope Aditya set. "Beyond scope" is not a gap in the workshop - it is content that belongs to a more technical audience, kept visible so nobody thinks it was missed. The source videos are referenced here for planning only and are not named anywhere in the workshop materials themselves.
