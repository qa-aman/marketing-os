Package what you repeat

Think about the instruction you have typed more than once this week. Same steps, same context, same structure, retyped from memory each time. One missed detail and the output drifts.

A skill ends that. It is a saved workflow that Cowork runs on command, with the same quality every time you run it. A plugin goes one step further: it bundles several skills together so you can hand your whole toolkit to a colleague in one install.

Why this matters

You do the workflow once, carefully. Cowork repeats it on command after that. Same prompt, same quality, no retyping.

STEP 01

What a skill is

A skill is a file called SKILL.md with two parts: a name, and a description. The description is the part that matters most.

The two parts

  • Name - for humans. It is how you and your team refer to the skill.
  • Description - for Cowork. This is the trigger. Cowork reads it to decide when to load and run the skill, without you asking for it by name.

Here is what a real one looks like:

weekly-content-brief/SKILL.md 
name: weekly-content-brief
description: Use when asked for the weekly Verdal content brief.

Workflow:
1. Read context/ for Verdal brand rules, voice, and ICP.
2. Pick 3 topics for Priya (the content lead) based on
   what is happening in plant care this week.
3. Write the brief in Verdal's voice.
4. Save it to projects/.
Why this matters. The description is not a label, it is a trigger. Get it right and the skill fires automatically the next time you ask for something that matches it. Get it vague and Cowork will not know to reach for it.
STEP 02

Three ways to create a skill

You do not need to start from a blank page every time.

  1. Extend an official skill - start from an Anthropic template and adapt it.
  2. Package a workflow you already do - the fastest path, and the most common one. This is what you will do in the lab.
  3. Build a fresh one - a blank canvas, your own steps, for something nobody has templated yet.

Do this first

Open Cowork and go to Customize > Claude Skills. Add the skill-creator skill if it is not already there. This is the one skill you need before you can build any others: describe what you want packaged, and it writes the SKILL.md for you.

Most people in the room will use option 2. You already have a workflow you repeat. Today you just package it.

Build a skill

Turn the Verdal weekly content brief into a reusable skill. Two ways to see it: ask Cowork to build it (Method A), or read the result it saves (Method B).

Result
Method B - the saved SKILL.md 
name: weekly-content-brief
description: Use when asked for the weekly content brief.

Workflow:
1. Read context/ for brand rules, voice, and ICP (works for
   any brand folder, not just Verdal).
2. Pick 3 topics based on what is relevant this week.
3. Write the brief in the brand's voice.
4. Save it to projects/.

What good looks like

  • Cowork reads your context files before it writes the skill, not just before it runs the workflow.
  • The SKILL.md has a clear name and a description that reads like a trigger, not a title.
  • Nothing brand-specific is hardcoded inside the skill. It says "read context/", it does not paste Verdal's voice rules directly into the file.
  • A Save Skill button appears. Click it, then open the saved skill and check the name and description.

Test it

Next time, do not retype the workflow. Just ask for the weekly content brief in a few words and see if Cowork reaches for the skill on its own. If it does not fire, the description was too vague, go back and make it more specific about when to use it.

STEP 04

Plugins

A plugin bundles skills, so you can share your whole toolkit in one install.

A plugin is Skills (every workflow you have packaged) plus Connectors (linked tools and data sources) plus Helpers (shared context files and templates). A new teammate installs one package and gets everything, no setup, no retraining.

Why this matters. Skills are for you. Plugins are for the team. Once you have packaged a few skills you use often, bundling them into a plugin is how you hand your whole Verdal toolkit to someone new in one step.
