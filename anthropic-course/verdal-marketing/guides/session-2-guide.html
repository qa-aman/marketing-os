Make Cowork yours: standing context and projects

Last session you handed Cowork a task and it saved a finished file. But you still had to explain your brand before it could start. Today you set that up once: a short set of rules Cowork reads before every task, and a project that remembers its own brand, its own instructions, and its own past work. By the end you will never re-explain your brand to Cowork again.

Worked brand: Verdal, a plant care brand. Dataset: verdal-marketing/. Official course: Introduction to Claude Cowork.

Why standing context

Without standing context, every Cowork session starts blank. You paste your brand rules, your folder conventions, your voice, and Cowork re-learns what it already knew yesterday. Multiply that across every task, every week, and it adds up to a lot of repeated typing for no reason.

Standing context fixes this two ways:

  • Global instructions - a short block of rules Cowork reads before every task, in every session, no matter the folder.
  • Projects - a reusable workspace with its own instructions, its own context, and a memory of everything it has already made.

They work together. Global rules still apply inside a project. A project just adds a second, more specific layer on top, for one workflow you run again and again.

STEP 01

Global instructions

A short block of text you write once in Settings. Cowork reads it before every task, in every session, no matter which folder you point it at. Think of it as a standing briefing, not a per-task paste.

Do this

  1. Open Claude Desktop, click the gear icon (bottom left) for Settings.
  2. Go to Settings > Cowork > Global instructions.
  3. Type your rules into the one text field, then save. One or two lines is enough. Save the detail for project instructions.
example global instructions 
Always read context/ before any task. Save output to projects/.
Match brand-voice.md. No em dashes, no emojis.
Keep it short. Under 100 words is plenty. Only put in what should apply to everything: voice, folder conventions, output format, what to avoid. Anything specific to one workflow belongs in a project's own instructions, not here.

Test it

Hand Cowork any small task without pasting brand rules. If it still reads context/ and matches your voice on its own, the global instruction is working.

STEP 02

Projects

A project is a named workspace inside Cowork. It has its own folder, its own instructions, and its own memory of past work. Unlike global instructions, a project is built for one repeatable workflow, not everything you do.

What a project remembers

  • The folder you pointed it at, and everything inside it.
  • The project instructions you wrote for it.
  • Reference examples you loaded in.
  • Every piece of work it has already produced.
  • Feedback you gave it on past runs.

Open the project again next week and none of that is gone. It still knows the brand, the design rules, and what it made last time, the same way a dedicated Slack channel keeps its pinned context and history for one campaign.

Global vs a project, in one line. Global instructions are the rules every new colleague reads on day one. A project is the dedicated workspace for one recurring job, with its own memory.
STEP 03

Build a Verdal carousel project

The hands-on build. Turn the carousel-design-system folder into a real project, with its brand context and reference examples already loaded, so every future carousel run starts here.

Do this

  1. In Cowork, create a new project from your carousel-design-system folder.
  2. Confirm it can see context/ (brand files) and reference/ (example carousels).
  3. Paste the prompt below.
Paste into Cowork
build the carousel project 
Create a project from my carousel-design-system folder.
Read the reference examples and brand context, then write
a brand style guide and a carousel design system file, and
save them in the project.

Pick one, so everyone's output is different:

  • A carousel project from the carousel-design-system folder
  • A weekly-email project
  • A competitor-watch project

What good looks like

  • Cowork reads reference/carousel-examples.md and context/ before writing anything.
  • A brand-style-guide.md and a carousel-design-system.md land in the project's output folder.
  • Both files sound like Verdal: warm, expert, plain, with green and terracotta named as the palette.
  • You did not paste a single brand detail into the prompt. It came from the folder and your global instructions.

Test it

Close the project, reopen it, and ask it to draft one more carousel slide. If it still follows the design system without you re-explaining it, the project's memory is working.

Set it up once, reuse forever. This project now has everything a carousel run needs: brand context, design rules, and reference examples. Next time, skip the setup and just ask for the carousel.
STEP 04

Global instructions vs projects

Same idea, standing context, but a different scope. Use this table to decide which one a new rule belongs in.

Global instructionsProjects
ScopeEvery task, every session, every folderOne repeatable workflow
Where it livesSettings > Cowork > Global instructionsIts own folder, with its own instructions file
What it holdsVoice, folder conventions, output format, what to avoidBrand context, reference examples, past output, feedback
Good example"No em dashes, no emojis, save to projects/""Verdal carousel", with its own design system and history
Use both together. Global rules for what should always be true. A project for the workflow you run again and again. Global instructions still apply inside a project.

Check yourself

Try these a day or two after the session, answers below.

Q: What is the difference between global instructions and a project?
A: Global instructions apply to every session and every folder, a project is one reusable workspace with its own instructions and memory.
Q: Where do you set global instructions?
A: Settings > Cowork > Global instructions.
Q: What does a project remember between sessions?
A: Its brand context, project instructions, reference files, and every piece of output it has produced.
