Claude Cowork for marketers

Verdal recipe library

Six real marketing jobs, each as one copy-paste prompt. Built for the Verdal kit, so the file paths are already correct. Change the topic, run it, review the output.

How to use these. Open Cowork with your verdal-marketing folder loaded. Copy a prompt, paste it, and let Cowork work. Everything lands in projects/. The output is a strong draft, not a finished post, so read every file before you use it. Once a recipe works for Verdal, swap the paths for your own brand folder and it works there too.
RECIPE 1 Cowork only

Repurpose one post into 7 assets

Turn a single blog post into three Instagram captions, a newsletter section, and three LinkedIn posts, all in Verdal's voice, in one run. Drop your source post into projects/repurpose/input/ first (name it source-post.md).

paste into cowork 
Read context/brand-voice.md and context/ideal-customer-profile.md.
Then read projects/repurpose/input/source-post.md.

Using only the ideas in that post, generate:

1. Three Instagram captions. Each: a one-line hook (not a question),
   one concrete tip from the post, a soft CTA to the Care Box or a
   product, 100-150 words, warm and expert tone.

2. One newsletter section: a two-sentence intro naming the problem,
   three specific bullet takeaways, one closing line linking to the post.

3. Three LinkedIn posts: one educational, one story-led, one contrarian.
   150-200 words each. Professional but not stiff.

Save to projects/repurpose/output/ as:
- instagram-captions.md
- newsletter-section.md
- linkedin-posts.md
Good looks like: captions that name a specific number or signal, a newsletter section that reads like a real email, and three LinkedIn posts that actually feel different from each other. If one format is off, say so in the same chat and ask for a rewrite of just that file.
RECIPE 2 Cowork only

Carousels at scale from the planner

Produce a full on-brand carousel for every topic in your planner, in one pass, using the Verdal design system. The planner ships in your kit at data/content-planner.csv.

paste into cowork 
You are running the Verdal carousel production workflow.

Load first:
- templates/carousel-design-system.md (visual and structural rules)
- context/brand-voice.md (tone and language)
- context/ideal-customer-profile.md (audience)
- data/content-planner.csv (the production queue)

Task:
1. Read every row in the planner where format is "carousel" and
   status is "planned".
2. For each, generate a carousel following the design system exactly:
   a hook slide, a context slide, four content slides, a CTA slide.
3. Save each as projects/output/carousels/.md, with slide
   number, headline, body copy, and a one-line image direction per slide.
4. Return a summary table: topic, status, and output path for each.

Voice: warm, expert, plain. No hype. Colors: botanical green, cream,
terracotta. Reference those colours in the image direction notes.
Good looks like: a hook slide that names the pain in under eight words, each middle slide doing exactly one thing, and image notes specific enough to hand to a designer ("overhead shot of a waterlogged pot on cream linen" beats "nice plant photo").
RECIPE 3 Cowork only visuals need an image connector

Ad creative briefs for five campaigns

Generate a structured creative brief per campaign: audience insight, three hooks, primary text, headline, CTAs, and visual direction. The brief is the single source of truth every ad traces back to. List your campaigns in a file at projects/ads/ad-campaigns.md (campaign, product, goal, platform, audience).

paste into cowork 
You are a paid social strategist for Verdal.

Read context/brand-context.md, context/brand-voice.md, and
context/ideal-customer-profile.md.
Then read projects/ads/ad-campaigns.md and read each campaign row.

For each campaign, write a creative brief and save it to
projects/ads//brief.md. Each brief includes:
- Campaign overview: product, goal, platform, audience (2-3 sentences)
- Audience insight: one specific pain or desire this campaign addresses
- Hook options: 3 hooks under 10 words (curiosity, result, problem)
- Primary text: 50-80 words, leads with the hook, ends with the CTA
- Headline: under 7 words, benefit first
- CTA text: 3 options
- Visual direction: scene, mood, colour cues in 2-3 sentences

Voice: warm, expert, plain. No corporate speak. No em dashes.
Work through all campaigns in parallel. Confirm when every brief is saved.
Good looks like: a brief whose audience insight names a real, specific pain, not "wants healthy plants". The visual direction step (generating actual images) needs an image connector, which you set up in the connectors session. Until then, the briefs alone are usable.
RECIPE 4 Cowork only publishing needs a CMS connector

SEO blog draft, structured for search

Write a full, structured SEO post in Verdal's voice from a target keyword, ready for a human to review and publish. Pick a keyword from data/keyword-opportunities.csv.

paste into cowork 
Read context/brand-context.md and context/brand-voice.md.
Read data/keyword-opportunities.csv and pick the highest-opportunity
keyword we have not written yet.

Write a complete SEO blog post for Verdal on that keyword:
- Title: 55-65 characters, includes the keyword
- Meta description: 145-155 characters, one clear promise
- Intro: 80-100 words, answers the core query in the first paragraph
- Table of contents: linked H2 headings
- Body: 4-6 H2 sections, 150-250 words each, bullets where they help
- Key takeaways box near the end: 3-5 scannable bullets
- Conclusion: 60-80 words, no new info, soft CTA

Use the keyword naturally, not forced. Voice: warm, practical, plain.
Save as projects/seo-blog/DRAFT-.md.
Report word count and where the keyword appears.
Good looks like: the intro answers the question in the first two sentences, the keyword sits in the title and first paragraph, and no sentence sounds like "maximise your botanical investments". Pushing the draft into a CMS like WordPress is a connector step for later; the draft itself needs nothing but Cowork.
RECIPE 5 Cowork only

Weekly competitor read

Turn the competitor data in your kit into a short, useful read: what each rival posted, what worked, and one thing Verdal should try. Uses data/competitor-posts.csv, which ships in the kit.

paste into cowork 
Read data/competitor-posts.csv and context/brand-context.md.

Write a one-page competitor digest for the Verdal marketing lead:
1. A three-bullet summary: what The Sill, Bloomscape, and Leon and
   George each leaned into this period.
2. The single highest-engagement post across all competitors, with the
   format and a one-line guess at why it worked.
3. Two content ideas for Verdal inspired by the gaps, each with a format
   (carousel, reel, single) and a one-line hook.
4. One thing to avoid, based on a post that clearly underperformed.

Keep it under 300 words. Plain language, no hype.
Save to projects/competitor-digest.md.
Good looks like: ideas tied to a real number from the data, not generic "post more reels". This is the recipe to put on a schedule later (see Recipe 6) so it lands every week on its own.
RECIPE 6 Cowork only

Schedule the whole engine

Once a recipe works by hand, put it on a repeat so it runs without you. In Cowork, type /schedule, set a recurrence, and paste an instruction like the one below. Test the recipe manually once before you schedule it.

paste when cowork asks what to run 
Run the weekly Verdal content check.

1. Read data/content-planner.csv. Find every row where status is "planned".
2. For each, load context/brand-voice.md and the relevant audience file,
   then draft the piece named in the topic column, in its format.
3. Save each draft to projects/output/ named
   DD-MM-YYYY--.md
4. Update that row: set status to "draft ready" and fill output_file.
5. Write a short run summary: how many drafted, anything skipped and why,
   anything that needs a human eye.

Do not publish anything. Draft and update the planner only.
One thing to know. Cowork scheduled tasks are not cloud jobs. They run only while Claude Desktop is open and the project is loaded. If your laptop is closed when the task is due, the run is skipped, it does not queue for later. Keep the app open, or set a reminder to open it.
Good looks like: you open your laptop and the week's drafts are already in projects/output/ with the planner updated, waiting for your review. Automation does the production. You keep the sign-off.

These recipes read your brand context at runtime, so they stay on-brand without you re-explaining Verdal each time. Never paste brand rules directly into a prompt, keep them in context/ so there is one source of truth. The one-page cowork-cheat-sheet.html in this folder is the quick reference that sits behind all of these.
