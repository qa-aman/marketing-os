Claude Cowork workshop - before Session 1

Session 0 - Get set up

Fifteen minutes on your own, before we meet. Do this and Session 1 starts with real work instead of installs.

Why this matters. Cowork is only as good as the folder you give it. If everyone arrives installed, signed in, and pointing at the Verdal folder, we spend Session 1 building a strategy deck, not watching a progress bar. If you get stuck, message the group before the session, not during it.

What you need

  • A Claude account on a plan that includes Cowork, the desktop app (not just the browser chat).
  • A laptop, Mac or Windows.
  • The Verdal kit folder (the facilitator will share it, a folder called verdal-marketing).
  • Fifteen minutes.
Read this first: can you install apps? This workshop needs the Claude desktop app on your machine. If your work laptop is locked down or IT has to approve installs, request approval now so it clears before Session 1. If you are not sure, ask IT today. This is the single most common reason someone cannot take part on day one.
STEP 1

Install Claude Desktop with Cowork

Download Claude Desktop from your account, sign in, and confirm you can see Cowork in the left sidebar. Cowork is the mode that works with local folders on your computer. Chat is the cloud mode that only hands you text. We use Cowork the whole way through.

If Cowork is not in your sidebar, your plan may not include it yet. Flag this in the group so the facilitator can sort it before we start.

STEP 2

Put the Verdal kit on your computer

Save the verdal-marketing folder the facilitator shared somewhere easy to find, like your Desktop or Documents. Do not rename it. Inside it you should see these folders:

verdal-marketing/ 
context/     brand knowledge Cowork reads before every task
templates/   reusable structures (deck outlines, carousel rules)
data/        sample spreadsheets (metrics, competitors, planner)
projects/    example finished work
guides/      these guides, including this one
brand-photos/ real product photos to use in demos

This is the sandbox for every demo and lab. It is fictional and safe to experiment in, so you can try anything without touching real client work.

STEP 3

Point Cowork at the folder

Open Cowork. Click the folder icon in the top left, find your verdal-marketing folder, and select it. Cowork now reads and saves files inside that folder. When you open a new task, this is the workspace it works in.

Nothing to type yet. You just want to confirm you can point Cowork at the folder and see the files listed.

STEP 4

Set your global instructions

In Claude Desktop, open Settings > Cowork > Global Instructions. These are rules that apply to every session, set once. Keep them short. Too much detail here just boxes Claude in.

paste this 
Always scan for a context/ folder before starting a task and read
everything in it. Save finished work into the projects/ folder.
Use sentence case, no em dashes, no emojis.

Two lines is plenty. Save and close.

What good looks like

  • Cowork appears in your sidebar and you can open it.
  • You confirmed you can install apps on your laptop, or IT approval is in progress.
  • Your verdal-marketing folder is on your computer with context/, templates/, data/, and projects/ inside.
  • You pointed Cowork at that folder and can see the files.
  • Your global instructions are saved and one or two lines long.

Common mistakes

  • Using Chat instead of Cowork. Chat cannot read or save to your folders. Make sure you are in Cowork before Session 1.
  • Leaving the install too late. If IT has to approve it, a same-day request will not clear in time. Ask now.
  • Writing a novel in global instructions. Long rules make Claude rigid. Keep it to a couple of lines.
  • Renaming or reorganising the kit folder. The demos and labs reference these exact folder names. Leave them as they are.

Bring your laptop set up like this to Session 1. If anything above did not work, say so in the group before we meet, with a screenshot if you can, so we can fix it ahead of time. The one-page cowork-cheat-sheet.html in this folder is a good next read once you are installed.
