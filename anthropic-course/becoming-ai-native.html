BECAUSE - Cowork course review

From a great workshop to an AI-native team

A deep review of the whole Cowork course through four lenses: learning design, adoption, content, and operating it inside BECAUSE. The materials are top tier. The opportunity is everything around them. This is how you make it 10x.

Reviewed: the six sessions (decks, guides, facilitator scripts), the design docs, and the Verdal kit. This file lives outside the kit, so it is not shipped to the team.

The one sentence. You have built one of the best Cowork training packages I have seen. It is also engineered to end at Session 6, on a fictional brand, with no way to know if anyone changed how they work. A workshop makes people aware. A system makes them AI-native. You have the workshop. The 10x is the system around it.

The diagnosis

Four independent reviews converged on the same thing. The craft is not the problem. Every session has a deck, a step-by-step guide, a facilitator script, and a real dataset to practice on. That is rare and genuinely good.

The problem is that the course is structured to stop at the exact moment it would start to matter. Five of six sessions run entirely on a fictional plant-care brand. There is no assessment, so no one knows what landed. Homework is never checked. The last touchpoint is Session 6, after which the training literature says an un-reinforced skill decays in two to four weeks. And nothing produces a lasting team asset: no shared skill library, no owner, no measure of whether it worked.

So the gap is not "teach more Cowork". It is the distance between the room and Monday morning. Close that distance and the same content produces an AI-native team instead of twelve people who once attended a good workshop.

What is already strong (keep these)

  • The session shape is a real scaffold. Recap, teach, demo, break, lab, debrief, homework, identical every week. Non-technical learners spend their attention on the content, not on decoding the format.
  • Verdal removes the blank-page problem. A complete, realistic, pre-built dataset means labs fail on prompt quality, not on missing setup. Learning on safe fake data before real data is the right sequence.
  • Four polished artifacts per session, all built. 24 files, consistent format, minute-by-minute run of show, and a "if the demo fails, do X" fallback. This is a real deliverable, not an outline.
  • It mirrors Anthropic's official Cowork course. Credibility with skeptics, plus a certificate path.
  • Safety is taught as a habit, not a rule. "Read what changed, open it, approve or redirect" is a behavior, not a compliance checkbox.
  • Session 6 already reaches for the bridge. The closing "write a prompt for one real task you own" is the single best adoption instinct in the design. It just needs to be pulled forward and followed up.

Fix before you deliver (hygiene, not strategy)

These are small but would embarrass a live delivery. Do them first, they are quick.

Broken pointers in the facilitator guides. The guides tell the facilitator to "follow the demo script (see session-1-demo-script.md, to be built in Phase 3)" and "the lab worksheet (Phase 3 deliverable) has the exact steps". Those files do not exist. A facilitator following the script live hits a dead reference. The phrase appears about five times across Sessions 1 and 4. Either build those two documents (most of the content is already inline) or replace the pointers with the inline steps.
The kit's connectors-setup.md is the wrong document, and it is now inside the kit you are shipping. Sessions 4 and 5 link to it for setup help, but it is a setup hub for a much broader Claude Code course: it covers install scripts, MCP, sub-agents, n8n, Gemini, and Perplexity, tools this course explicitly puts out of scope ("we start with Cowork, not Claude Code"). An attendee who opens it finds instructions for a tool the course says not to use. Rewrite it scoped to only what this course needs: Google Workspace, Microsoft 365, Claude in Chrome, and the handful of connectors the demos use.
A hardcoded model name and unverified UI paths. Session 1's deck shows "Opus 4.8" in the model-picker mock. Genericize it, a wrong version number on slide one dents credibility. Session 5's own facilitator guide flags two UI locations (the pin button, the scheduled-tasks panel) as unverified. Every specific menu path (Settings > Cowork, Customize > Connectors, Preferences > Features) goes stale fastest. Add a standard "verify these paths the morning of" line, and check them in the dry run.

If you only do five things

Ranked by leverage. Each is drawn from a different lens, and together they close all five gaps.

  1. Build a real because-marketing/ kit and use it in the room. Mirror Verdal's structure exactly with BECAUSE's real brand voice, ICP, and one real template. This is the single point where "we did an exercise" becomes "we have a working system."
  2. Add the retention loop. A 3 to 5 question retrieval quiz sent after every session (not just Session 6), and homework that gets dropped in a shared channel and gets one line of feedback before the next session.
  3. Name an owner and stand up a shared skill library. One person owns keeping it current, triaging new skills, and onboarding new joiners. Every skill built lands in one shared, named place, not a Slack DM.
  4. Pull the manager in, and add a 30-day graduation session on real work. Managers get a 15-minute brief and one question to ask in 1:1s. Thirty days out, the team reconvenes with the real task they actually ran, not Verdal.
  5. Write real client-data rules, and teach how to check AI output. One page: what data may enter a Cowork session, per-client isolation, who signs off. Plus a 10-minute segment on catching a confident-but-wrong claim before it ships.

The rest of this report is the full move set behind these five, grouped by the outcome each drives.

The AI-native maturity ladder

"10x" is not a bigger workshop. It is moving the whole team up this ladder. The course as built gets people to Level 1. The moves above get them to Level 3.

Level 0

Attended

Sat through six sessions. Can describe what Cowork does. No behavior change guaranteed. This is where an un-reinforced workshop leaves people.

Level 1

Can do it on demand

Can run a task on Verdal, and with help on their own brand. Where the current course lands its best attendees.

Level 2

Uses it by default on real work

Reaches for Cowork first on the workflows in the backlog. Pulls from and contributes to the shared skill library. Their manager expects it. This is what the retention and adoption moves buy.

Level 3

AI-native

The team's default operating system. New work is scoped assuming AI does the first draft. The library compounds, new joiners onboard into it, and time saved is measured and reinvested. Self-sustaining without the instructor.
